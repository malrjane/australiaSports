WEBSITE DESIGN SPECIFICATION

THEME & VISUAL TOKENS

1.  **Color Palette (Slate-Indigo-Cyan):**
    *   **Backgrounds:** Slate-50 (main surface), Slate-100 (secondary sections), White (cards).
    *   **Text:** Slate-900 (primary headings), Slate-600 (body copy), Indigo-600 (links/accents).
    *   **Primary Action:** Indigo-600 (hover: Indigo-700).
    *   **Accents:** Cyan-400 to Indigo-500 gradients for subtle borders or overlays.
    *   **DNA:** Vertical stripes pattern (using subtle CSS repeating-linear-gradient on section backgrounds) to create rhythm.

2.  **Typography (Brutalist / Inter):**
    *   **Font:** Inter (variable weight).
    *   **Headings:** Bold to Black weights. Tight tracking (letter-spacing: -0.05em). Uppercase for H1/H2 to lean into the "Poster/Afisha" vibe.
    *   **Scale:** Massive responsive scaling. H1 on mobile text-4xl -> desktop text-7xl.
    *   **Body:** High legibility, text-base to text-lg.

3.  **Shapes & Elevation:**
    *   **Radius:** Segmented-round. Buttons are full pill-shape (rounded-full). Cards have distinct rounded corners (rounded-2xl).
    *   **Shadows:** Flat or subtle. Emphasis is created via borders (border-slate-200) and layout rather than deep drop shadows.
    *   **Spacing:** "Hyper-minimal" internal spacing (tight gaps between text and subheads) contrasted with generous "Boxed" padding around the outer edges of containers.

4.  **Layout & Motion:**
    *   **Container:** "Boxed-content" approach. Max-width-7xl centered, with distinct x-padding (px-6 md:px-12).
    *   **Grid:** Strict column alignments.
    *   **Motion:** "Shake-soft". Buttons and cards have a subtle wiggle or scale-up on hover/focus.
    *   **Breakpoints:** Mobile-first. Aggressive layout changes at 'md' and 'lg'.

IMAGE GUIDELINES
*   **Source:** Unsplash / Pexels.
*   **Queries:** "Marathon runner", "Triathlon bike", "Sydney Opera House distance", "Melbourne sports stadium", "Outback road", "Swimmer in ocean".
*   **Treatment:** Architectural, high contrast. Use `object-cover` for fills.
*   **Lazy Loading:** All images below the fold must have `loading="lazy"`.
*   **Restrictions:** NO small thumbnails. If a visual element is small (<400px), use a Feather Icon or SVG shape.

ACCESSIBILITY & PERFORMANCE
*   **Contrast:** Ensure Slate-600 text on Slate-50 background meets AA standards.
*   **Focus:** All interactive elements need a visible `focus:ring` outline (Indigo-400).
*   **Tags:** Use semantic `<header>`, `<main>`, `<footer>`, `<nav>`, `<section>`.
*   **Forms:** All inputs must have associated `<label>` elements.
*   **Preconnect:** Suggest preconnecting to image providers.
